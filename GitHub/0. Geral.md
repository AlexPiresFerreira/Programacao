
## 1. Conceito geral

- **GitHub** é uma plataforma para:
  - Hospedar repositórios Git (código, documentação, arquivos).
  - Colaborar em equipe (issues, pull requests).
  - Automatizar tarefas (actions).
  - Organizar projetos (projects, wiki).
  - Publicar snippets de código (gists).

---

## 2. Recursos principais do repositório

### 2.1 Issues
- Servem para **registrar tarefas, bugs, ideias e melhorias**.
- Você pode:
  - Atribuir **pessoas responsáveis**.
  - Definir **labels** (tags de categoria, prioridade etc.).
  - Usar **milestones** (entregas/versões).
- Funcionam como **“tickets”** do projeto.

### 2.2 Pull Request (PR)
- Usado para propor **alterações de código**.
- Fluxo comum:
  - Você cria uma **branch** com a nova versão/melhoria.
  - Envia um **pull request** para o repositório principal.
  - O dono/manutentor revisa, comenta e aprova (ou pede mudanças).
- Ideal para:
  - Revisão de código.
  - Discussão antes de mesclar mudanças.

### 2.3 Actions
- Sistema de **automação** do GitHub (CI/CD).
- Permite criar workflows para:
  - Rodar testes automaticamente a cada commit/pull request.
  - Fazer build, deploy, lint, etc.
- Você configura arquivos `.yml` na pasta `.github/workflows`.

### 2.4 Projects
- Ferramenta de **gerenciamento de projeto** dentro do GitHub.
- Serve para:
  - Criar **boards** (Kanban, Scrum etc.).
  - Organizar issues e PRs em colunas (por exemplo: *Backlog → Em andamento → Concluído*).
- Ajuda a ter uma visão de **pipeline** do projeto.

### 2.5 Wiki
- Espaço de **documentação do repositório**.
- Usado para:
  - Descrever arquitetura.
  - Registrar processos, tutoriais, guias de contribuição.
- Cada página é um arquivo Markdown.

### 2.6 Security
- Área para gerenciar a **segurança do repositório**.
- Exemplos:
  - **Política de segurança** (arquivo `SECURITY.md`).
  - Alertas de vulnerabilidade em dependências.
  - Code scanning / secret scanning.

### 2.7 Insights
- Exibe **informações detalhadas do projeto**, como:
  - Gráfico de commits.
  - Contribuidores.
  - Issues abertas/fechadas.
  - Activity, traffic, forks, dependabot alerts etc.
- Útil para entender a **evolução** e o engajamento do repositório.

### 2.8 Settings
- Área de **configuração do repositório**.
- Exemplos:
  - Nome, descrição, visibilidade (público/privado).
  - Proteção de branches.
  - Webhooks, integração com outros serviços.
  - Configuração de Pages, Actions, segurança etc.

---

## 3. GitHub Gist

- **GitHub Gist** é um “bloco de anotações” de código.
- Serve para:
  - Compartilhar snippets, exemplos, scripts pequenos.
  - Criar notas rápidas com sintaxe de código.
- Pode ser:
  - **Público** (qualquer pessoa vê).
  - **Secreto** (não aparece listado, mas quem tem o link vê).
- Bom para:
  - Trechos de código que não precisam de um repositório inteiro.
  - Anotações técnicas reutilizáveis.
