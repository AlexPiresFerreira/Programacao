

- `try` → bloco onde o código que pode gerar erro é executado.
- `except` → bloco que é executado caso um erro ocorra.
- O programa **não quebra** quando o erro é capturado pelo `except`.

### Sintaxe básica

```python
try:
    # código que pode gerar erro
except:
    # o que fazer se der erro
```

### Exemplo básico

```python
try:
    preco = float(input("Informe o preço: "))
    print(f"Preço informado: {preco}")
except:
    print("Valor inválido! Digite um número.")
```

---

## Capturando erros específicos

- Podemos capturar tipos específicos de erro para tratar cada caso.

```python
try:
    numero = int(input("Informe um número: "))
    resultado = 10 / numero
    print(f"Resultado: {resultado}")
except ValueError:
    print("Erro: valor inválido. Digite apenas números.")
except ZeroDivisionError:
    print("Erro: não é possível dividir por zero.")
```

---

## `try/except/else/finally`

### Estrutura completa

```python
try:
    # código que pode gerar erro
except TipoDeErro:
    # executado se o erro ocorrer
else:
    # executado se NÃO houver erro
finally:
    # executado SEMPRE, com ou sem erro
```

### Exemplo completo

```python
try:
    numero = int(input("Informe um número: "))
    resultado = 10 / numero
except ValueError:
    print("Erro: valor inválido!")
except ZeroDivisionError:
    print("Erro: divisão por zero!")
else:
    print(f"Resultado: {resultado}")
finally:
    print("Execução finalizada.")
```

---

## Erros mais comuns em Python

| Erro | Quando ocorre |
|------|--------------|
| `ValueError` | Conversão de tipo inválida (ex: `int("abc")`) |
| `ZeroDivisionError` | Divisão por zero |
| `TypeError` | Operação com tipos incompatíveis |
| `IndexError` | Acesso a índice inexistente em lista/tupla |
| `KeyError` | Acesso a chave inexistente em dicionário |
| `AttributeError` | Uso de método/atributo que não existe no objeto |
| `FileNotFoundError` | Arquivo não encontrado |

---

## Boas práticas

- **Sempre especifique o tipo do erro** no `except` — evite `except` genérico.
- Use `else` para separar o código de sucesso do código de tratamento de erro.
- Use `finally` para fechar arquivos, conexões ou liberar recursos.
- Não use `try/except` para esconder erros ou substituir validações.

```python
# ruim → captura qualquer erro sem distinguir
try:
    resultado = int(input("Número: "))
except:
    print("Deu erro.")

# bom → captura erro específico e informa o usuário
try:
    resultado = int(input("Número: "))
except ValueError:
    print("Digite apenas números inteiros.")
```

### Boas práticas

- Prefira tratar erros específicos.
- `except Exception as erro` é útil para mostrar a mensagem real do problema.
- Evite usar `except:` sem necessidade.
