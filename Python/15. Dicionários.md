
## 1. Resumo rápido
- Dicionários são estruturas de dados que armazenam pares **chave → valor**.
- Permitem acesso rápido ao valor a partir da chave.
- São **mutáveis** e não garantem ordem de inserção em versões mais antigas do Python (em Python 3.7+ preservam a ordem de inserção).

---

## 2. Conceito
- Um dicionário é uma coleção de pares `chave: valor`.
- Chaves:
  - Devem ser **imutáveis** (ex.: `str`, `int`, `float`, `tuple`).
  - Não podem se repetir (chave duplicada sobrescreve o valor anterior).
- Valores:
  - Podem ser de qualquer tipo (incluindo listas, outros dicts etc.).

---

## 3. Criando dicionários

```python
# forma literal
pessoa = {
    "nome": "Guilherme",
    "idade": 28,
    "profissao": "Programador",
}

# usando dict()
pessoa2 = dict(nome="Giovanna", idade=27)

# dicionário vazio
vazio = {}
vazio2 = dict()
```

---

## 4. Acessando valores

### 4.1 Acesso direto pela chave

```python
pessoa = {"nome": "Guilherme", "idade": 28}

pessoa["nome"]   # "Guilherme"
pessoa["idade"]  # 28
```

> Se a chave não existir, `pessoa["chave"]` levanta `KeyError`.

### 4.2 Método `get`

```python
pessoa.get("nome")            # "Guilherme"
pessoa.get("fone")            # None
pessoa.get("fone", "vazio")   # "vazio"
```

---

## 5. Adicionando e alterando itens

- Atribuições em chaves criam ou sobrescrevem valores.

```python
contato = {}

contato["nome"] = "Guilherme"
contato["email"] = "gui@example.com"
contato["telefone"] = "1111-2222"

contato["telefone"] = "9999-8888"  # altera valor
```

### 5.1 `update`

```python
contato = {"nome": "Guilherme", "email": "gui@example.com"}

contato.update({"email": "novo_email@example.com"})
contato.update(telefone="9999-8888")

# {"nome": "Guilherme", "email": "novo_email@example.com", "telefone": "9999-8888"}
```

---

## 6. Removendo itens

### 6.1 `pop`

- Remove e devolve o valor da chave.

```python
contato = {"nome": "Guilherme", "email": "gui@example.com"}

email = contato.pop("email")
# contato -> {"nome": "Guilherme"}
# email -> "gui@example.com"
```

### 6.2 `popitem`

- Remove o **último** item inserido (em Python 3.7+).

```python
contato = {"nome": "Guilherme", "email": "gui@example.com"}
contato.popitem()
# remove ("email", "gui@example.com")
```

### 6.3 `del`

```python
contato = {"nome": "Guilherme", "email": "gui@example.com"}
del contato["email"]
```

### 6.4 `clear`

```python
contato = {"nome": "Guilherme", "email": "gui@example.com"}
contato.clear()   # {}
```

---

## 7. Iterando dicionários

```python
contato = {"nome": "Guilherme", "email": "gui@example.com"}

# chaves
for chave in contato:
    print(chave)

# chaves e valores
for chave, valor in contato.items():
    print(chave, valor)
```

### 7.1 Métodos de iteração

```python
contato.keys()     # retorna uma visão das chaves
contato.values()   # retorna uma visão dos valores
contato.items()    # retorna pares (chave, valor)
```

---

## 8. Operações úteis

### 8.1 Verificando existência de chave

```python
contato = {"nome": "Guilherme"}

"nome" in contato      # True
"email" in contato     # False
```

### 8.2 `len`

```python
len(contato)   # quantidade de pares chave-valor
```

### 8.3 `copy`

```python
contato = {"nome": "Guilherme"}
copia = contato.copy()
```

---

## 9. Dicionários aninhados

- Um valor pode ser outro dicionário.

```python
contatos = {
    "guilherme": {
        "nome": "Guilherme",
        "email": "gui@example.com",
    },
    "giovanna": {
        "nome": "Giovanna",
        "email": "gio@example.com",
    },
}

contatos["guilherme"]["email"]  # "gui@example.com"
```

---

## 10. Erros comuns

- Acessar chave inexistente com `dict["chave"]` → `KeyError`.
- Esquecer que dicionários **não** são indexados por posição (use sempre chave).
- Modificar um dicionário enquanto itera sobre ele sem cuidado.

---

## 11. Boas práticas

- Use chaves descritivas, em `snake_case`:
  - Ex.: `"nome_completo"`, `"data_nascimento"`.
- Prefira `get()` quando a chave pode não existir.
- Para estruturas mais complexas, use dicionários aninhados de forma organizada.
