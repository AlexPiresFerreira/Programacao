
## Resumo rápido
- Tuplas são estruturas de dados muito parecidas com listas.
- A principal diferença: **tuplas são imutáveis** e listas são mutáveis.
- São usadas quando você **não deve alterar** os dados depois de criar.

---

## Conceito
- Tuplas são sequências ordenadas de elementos.
- Podem armazenar **qualquer tipo de objeto** (números, strings, listas, outras tuplas etc.).
- São criadas:
  - com a classe `tuple()`, ou  
  - colocando valores separados por vírgula entre parênteses.

---

## Criando tuplas

```python
frutas = ("laranja", "pera", "uva")     # tupla comum
letras = tuple("python")                # a partir de um iterável
numeros = tuple([1, 2, 3, 4])           # a partir de uma lista
pais = ("Brasil",)                      # tupla de 1 elemento (vírgula é obrigatória)
```

**Observações importantes:**
- `("Brasil")` → é apenas uma string.  
- `("Brasil",)` → é uma tupla com um único elemento.

---

## Acesso direto por índice

- Tuplas são sequências, então o acesso é feito por índice.
- A contagem começa em **0**.

```python
frutas = ("maçã", "laranja", "uva", "pera")

frutas[0]   # "maçã"
frutas[2]   # "uva"
```

---

## Índices negativos

- Sequências suportam **indexação negativa**.
- A contagem começa em **-1** (último elemento).

```python
frutas = ("maçã", "laranja", "uva", "pera")

frutas[-1]  # "pera"
frutas[-3]  # "laranja"
```

---

## Tuplas aninhadas

- Tuplas podem armazenar outras tuplas.
- Isso permite criar estruturas bidimensionais (como tabelas).

```python
matriz = (
    (1, "a", 2),
    ("b", 3, 4),
    (6, 5, "c"),
)

matriz[0]        # (1, "a", 2)
matriz[0][0]     # 1
matriz[0][-1]    # 2
matriz[-1][-1]   # "c"
```

---

## Fatiamento (slicing)

- Podemos extrair um conjunto de valores de uma sequência.
- Sintaxe geral: `tupla[start:stop:step]`

```python
tupla = ("p", "y", "t", "h", "o", "n")

tupla[2:]       # ("t", "h", "o", "n")
tupla[:2]       # ("p", "y")
tupla[1:3]      # ("y", "t")
tupla[0:3:2]    # ("p", "t")
tupla[:]        # ("p", "y", "t", "h", "o", "n")
tupla[::-1]     # ("n", "o", "h", "t", "y", "p")  # tupla invertida
```

**Lembretes:**
- `start` → índice inicial (inclusivo).
- `stop` → índice final (exclusivo).
- `step` → passo (quantos elementos pula).

## Iterando tuplas com `for`

### `for` simples

```python
carros = ("gol", "celta", "palio")

for carro in carros:
    print(carro)
```

### `enumerate` (índice + valor)

```python
carros = ("gol", "celta", "palio")

for indice, carro in enumerate(carros):
    print(f"{indice}: {carro}")
```

## Métodos da classe `tuple`

### `count()`

Conta quantas vezes um valor aparece na tupla.

```python
cores = ("vermelho", "azul", "verde", "azul")

cores.count("vermelho")  # 1
cores.count("azul")      # 2
cores.count("verde")     # 1
```

### `index()`

Retorna o índice da **primeira ocorrência** de um valor.

```python
linguagens = ("python", "js", "c", "java", "csharp")

linguagens.index("java")    # 3
linguagens.index("python")  # 0
```

### `len()`

Retorna o **tamanho** da tupla.

```python
linguagens = ("python", "js", "c", "java", "csharp")

len(linguagens)  # 5
```

---

## Diferenças principais: lista x tupla

- **Lista**
  - Mutável.
  - Sintaxe: `[1, 2, 3]`.
  - Ideal quando você **vai alterar** os dados (inserir, remover, modificar).

- **Tupla**
  - Imutável.
  - Sintaxe: `(1, 2, 3)`.
  - Ideal para dados **fixos** (por exemplo, coordenadas, configurações, valores constantes).

---

## Boas práticas

- Use tuplas quando:
  - Os dados não devem ser alterados.
  - Você quer mais segurança contra modificações acidentais.
- Use listas quando:
  - Você precisa inserir/remover/alterar elementos com frequência.
- Lembre: tuplas podem ser usadas como **chaves de dicionário** (são hashable se os elementos também forem imutáveis).
