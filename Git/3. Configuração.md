
Antes de usar o Git, é comum configurar nome e e-mail:

- `git config --global user.name "nome"`
- `git config --global user.email "e-mail"`
- `git config --global color.ui true`

## Estrutura básica de trabalho

- **Criar repositório local**
    
    - `git init`
- **Clonar repositório remoto**
    
    - `git clone <URL>`
- **Verificar status**
    
    - `git status`
- **Adicionar arquivos ao stage**
    
    - `git add <arquivo>`
    - `git add *`
- **Criar commit**
    
    - `git commit -m "mensagem"`
- **Enviar para o remoto**
    
    - `git push`

## Comandos mais usados

|Comando|Função|
|---|---|
|`git clone <URL>`|Clonar um repositório remoto|
|`cd`|Entrar em uma pasta|
|`git status`|Ver o estado dos arquivos no repositório|
|`git add <arquivo>`|Adicionar arquivo ao stage|
|`git add *`|Adicionar todas as alterações|
|`git commit -m "mensagem"`|Salvar alterações no histórico|
|`git push`|Enviar alterações para o repositório remoto|
|`git pull`|Baixar e atualizar alterações remotas|
|`git checkout <branch>`|Trocar de branch|
|`git checkout -b <branch>`|Criar e mudar para uma nova branch|
|`git branch`|Listar branches|
|`git merge <branch>`|Mesclar uma branch em outra|
|`git rebase <branch>`|Reaplicar commits sobre outra base|
|`git reset HEAD~1`|Voltar um commit|
|`git log`|Ver histórico de commits|
|`git tag <tag>`|Criar uma tag|
|`git push origin <branch> --tags`|Enviar branch e tags|
|`git init`|Inicializar um repositório Git|
|`mkdir <pasta>`|Criar pasta|
|`ls -la`|Listar arquivos detalhados|
|`touch <arquivo>`|Criar arquivo vazio|
|`ssh-keygen`|Criar chave SSH|


## Histórico e inspeção

O Git também permite analisar o histórico do projeto com opções como:

- `git log`
- `git log -p`
- `git log --stat`
- `git log --pretty=oneline`
- `git log --since=x.days`

Esses comandos ajudam a ver:

- quem alterou
- o que foi alterado
- quando foi alterado
- quantas mudanças houve
