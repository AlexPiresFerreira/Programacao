
## 1. O que é o Git

- **Git** é um sistema de controle de versão.
- Ele serve para:
    - registrar alterações;
    - comparar versões;
    - colaborar em equipe;
    - voltar para estados anteriores quando necessário.

---

## 2. Configuração inicial

Antes de usar o Git, é comum definir seu nome e e-mail globalmente  

### Definir nome e e-mail

- `git config --global user.name "nome"`
- `git config --global user.email "e-mail"`

### Outras configurações úteis

- `git config --global color.ui true`
- `git config --global core.editor vim`
- `git config --global merge.tool vimdiff`
- `git config --global core.excludesfile ~/.gitignore`

### Ver configurações

- `git config --list`
- `git --version`

### Arquivo de configuração

- As configurações globais ficam salvas no arquivo **`.gitconfig`** do usuário.

---

## 3. Autenticação com SSH

Para trabalhar com repositórios remotos, uma forma comum de autenticação é usar **chave SSH**  

### Gerar chave SSH

- `ssh-keygen`

### Ver chave pública

- `cat <chave>.pub`

### Ajustar o remoto, se necessário

- `git remote set-url origin <URL>`

---

## 4. Estrutura básica de trabalho

### Criar repositório local

- `git init`

### Clonar repositório remoto

- `git clone <URL>`

### Verificar estado

- `git status`

### Adicionar arquivos ao stage

- `git add <arquivo>`
- `git add *`
- `git add .`

### Criar commit

- `git commit -m "mensagem"`

### Enviar para o repositório remoto

- `git push`
- no primeiro envio, normalmente:
    - `git push -u origin main`

### Baixar alterações remotas

- `git pull`

---

## 5. Comandos mais usados

|Comando|Função|
|---|---|
|`git clone <URL>`|Clonar um repositório remoto|
|`cd`|Entrar em uma pasta|
|`git status`|Ver o estado dos arquivos|
|`git add <arquivo>`|Adicionar arquivo ao stage|
|`git add *`|Adicionar todas as alterações|
|`git commit -m "mensagem"`|Salvar alterações no histórico|
|`git push`|Enviar alterações para o remoto|
|`git pull`|Baixar e atualizar alterações remotas|
|`git checkout <branch>`|Trocar de branch|
|`git checkout -b <branch>`|Criar e mudar para uma nova branch|
|`git branch`|Listar branches|
|`git merge <branch>`|Mesclar uma branch em outra|
|`git rebase <branch>`|Reaplicar commits sobre outra base|
|`git reset HEAD~1`|Voltar um commit|
|`git log`|Ver histórico de commits|
|`git tag <tag>`|Criar uma tag|
|`git push origin <branch> --tags`|Enviar branch e tags|
|`git init`|Inicializar repositório Git|
|`mkdir <pasta>`|Criar pasta|
|`ls -la`|Listar arquivos detalhados|
|`touch <arquivo>`|Criar arquivo vazio|
|`ssh-keygen`|Criar chave SSH|

## 6. Histórico e inspeção

O Git permite analisar o histórico do projeto com vários níveis de detalhe  

### Comandos úteis

- `git log`
- `git log -p`
- `git log --stat`
- `git log --pretty=oneline`
- `git log --since=x.days`

### O que você consegue ver

- quem alterou;
- o que foi alterado;
- quando foi alterado;
- quantas mudanças houve.

---

## 7. Repositório remoto

### Ver repositórios remotos

- `git remote`
- `git remote -v`

### Adicionar remoto

- `git remote add origin <url>`

### Ver detalhes do remoto

- `git remote show origin`

### Renomear remoto

- `git remote rename origin curso-git`

### Remover remoto

- `git remote rm curso-git`

---

## 8. Branches

### Criar branch

- `git branch bug-123`

### Trocar para branch

- `git checkout bug-123`

### Criar e trocar ao mesmo tempo

- `git checkout -b bug-456`

### Voltar para branch principal

- `git checkout master`
- ou `git checkout main`, dependendo do projeto

### Fazer merge

- `git merge bug-123`

### Remover branch

- `git branch -d bug-123`

### Listar branches

- `git branch`
- `git branch -v`
- `git branch --merged`
- `git branch --no-merged`

---

## 9. Stash

O **stash** guarda mudanças temporárias sem precisar commitar  

### Comandos

- `git stash`
- `git stash list`
- `git stash apply`
- `git stash apply stash@{2}`
- `git stash branch meu_branch`

---

## 10. Desfazendo alterações

### Desfazer alteração local

- `git checkout -- meu_arquivo.txt`

### Desfazer alteração no stage

- `git reset HEAD meu_arquivo.txt`

---

## 11. Tags

### Criar tag

- `git tag vs-1.1`

### Tag anotada

- `git tag -a vs-1.1 -m "Minha versão 1.1"`

### Enviar tag

- `git push origin vs-1.2`
- `git push origin --tags`

---

## 12. Resumo prático do fluxo

1. Configurar nome e e-mail
2. Criar ou clonar o repositório
3. Verificar o status
4. Adicionar arquivos
5. Criar commit
6. Enviar para o remoto
7. Atualizar com `pull` quando necessário
