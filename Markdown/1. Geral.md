## Resumo rápido

- **Markdown** é uma linguagem de marcação leve para formatar texto de forma simples e legível.
- É muito usada no **Obsidian**, **GitHub**, documentação e anotações.
- Permite criar títulos, listas, links, tabelas, blocos de código, citações, imagens e muito mais.

---

## 1. Organização de notas

### Tipos de nota no método Zettelkasten

- **Fleeting note**
    
    - Nota passageira.
    - Serve para registrar pensamentos rápidos.
    - Normalmente vai para a **inbox**.
- **Literature note**
    
    - Nota baseada em leitura/consumo de conteúdo.
    - Deve ser escrita com **suas próprias palavras**.
- **Permanent note**
    
    - Surge da revisão da fleeting + literature.
    - É a nota definitiva, conectada a outras ideias.

### Estrutura do vault

- **Vault**: pasta/cofre local onde ficam as notas.
- Cada vault pode ter **tema** e **configurações** próprias.

### Fluxo de organização

- **Inbox**
    
    - Notas para revisar depois.
    - Idealmente ficam ali por até **2 dias**.
- **Archive**
    
    - Destino para notas que ficaram mais de **2 dias** na inbox.

## 2. Estrutura básica do texto

### Títulos

Markdown usa `#` até `######` para definir níveis de título.


```markdown
# H1
## H2
### H3
#### H4
##### H5
###### H6
```

### Uso comum

- **H1**: nome do projeto ou nota principal
- **H2**: assunto principal
- **H3+**: subtópicos

---

### Quebra de linha

Você pode quebrar linha de duas formas:

- com **2 espaços** no fim da linha
- com `<br>`

```markdown
Linha 1  
Linha 2

Linha 1<br>
Linha 2
```

---

### Caracteres de escape

Use `\` para escapar caracteres especiais.


```markdown
\*texto\*
```

---

### Comentário

Comentários não aparecem no render final.


```markdown
<!-- Seu comentário -->
```

---

### Separador de linha

Você pode criar uma linha horizontal com:


```markdown
---
***
___
```

---

### Colapsar conteúdo

Usando HTML:


```markdown
<details>
  <summary>Ver mais</summary>
  Texto escondido até abrir.
</details>
```

## 3. Destaque no texto

### Itálico


```markdown
*texto*
_texto_
```

### Negrito

```markdown
**texto**
__texto__
```

### Tachado


```markdown
~~texto~~
```

### Negrito + itálico

```markdown
***texto***
```

### Negrito + itálico + riscado


```markdown
~~_**texto**_~~
```

### Citação


```markdown
> texto
>> texto
```

---

## 4. Listas

### Lista ordenada


```markdown
1. texto
2. texto
3. texto
```

### Lista não ordenada


```markdown
- texto
* texto
+ texto
```

### Lista aninhada

- Use indentação de **4 espaços** ou `tab`.


```markdown
- Item 1
    - Subitem
```

### Lista combinada


```markdown
1. Item
    - Subitem
```

### Lista de definição


```markdown
Texto
: definição
```

### Lista de tarefas


```markdown
- [ ] tarefa pendente
- [x] tarefa concluída
```

## 5. Tabelas


```markdown
| Cabeçalho | Cabeçalho |
|-----------|-----------|
| Texto     | Texto     |
| Texto     | Texto     |
```

- Muito útil para comparações e organização de conteúdo.

---

## 6. Links e imagens

### Link


```markdown
[texto do link](URL "Título ao passar o mouse")
```

### Exemplo de link interno


```markdown
[texto do link](#pagina "Título ao passar o mouse")
```

### Link para arquivo


```markdown
[texto do link](arquivo.extensão "Título ao passar o mouse")
```

### Link interno do Obsidian


```markdown
[[Pasta ou Nota]]
```

### Imagem


```markdown
![texto alternativo](URL da imagem "Título ao passar o mouse")
```

### Badge / imagem clicável


```markdown
[![texto alternativo](URL da imagem "Título")](URL do link)
```

### Ajuste de tamanho

- Em alguns ambientes, pode ser feito com HTML ou extensão específica do editor.

## 7. Código

### Código em uma linha


```markdown
`nano`
```

### Bloco de código


````markdown
```bash
ls -la
```
````

### HTML


```markdown
<code>
Código
</code>
```

### Dica

- Use o nome da linguagem depois das crases para destacar a sintaxe.

## 9. Obsidian e recursos úteis

### Conceitos úteis

- **Tags**: `#tag`
- **Properties**: metadados da nota
- **Preview**: visualização da nota renderizada

### Plugins citados

- **Calendar**
     - Adiciona um **calendário** para navegar por notas diárias e acompanhar registros por data.
- **Iconize**
     - Permite adicionar **ícones personalizados** em notas, pastas e itens do cofre para organização visual.
- **File Color**
     - Serve para **colorir arquivos e pastas**, facilitando a identificação visual no explorador.
- **Colored Tags**
     - Deixa as **tags coloridas**, ajudando a diferenciar categorias e temas rapidamente.
- **File Explorer Note Count**
    - Mostra a **quantidade de notas/arquivos** em pastas no explorador de arquivos.
- **Code Styler**
    - Melhora a **estilização de blocos de código**, deixando o visual mais agradável e legível.
- **File Explorer++**
	-  Amplia os recursos do explorador de arquivos, trazendo **mais controle e organização** para pastas e notas.
- **Ninja Cursor**
    - Facilita a **navegação rápida pelo texto**, melhorando a movimentação do cursor e a edição.
- **Commander**
    - Permite criar **atalhos e comandos personalizados** para agilizar ações dentro do Obsidian.
- **Editor Syntax Highlight**
     - Destaca a **sintaxe de forma mais clara** no editor, melhorando a leitura de código e marcações.
- **Importer**
     - Ajuda a **importar conteúdo de outras ferramentas** ou formatos para dentro do Obsidian.
- **Tag Wrangler**
     - Serve para **organizar, renomear e gerenciar tags** com mais facilidade.
- **Tasks**
     - Adiciona suporte avançado a **listas de tarefas**, com filtros, consultas e acompanhamento de pendências.
- **Nested tags graph**
     - Mostra a relação entre **tags aninhadas** em forma de grafo, ajudando a visualizar hierarquias de assuntos.

### Tema

- **Obsidianite**
  - Tema visual para o Obsidian com foco em contraste e legibilidade.
